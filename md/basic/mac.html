# mac新手入门指南

mac电脑上非常适合程序员的电脑，建议工作后的程序员，如果公司不能给配置mac电脑，自己要争取跳槽发mac电脑的公司，或者自己花点钱买一个，绝对物超所值。

我在大学时买过多个电脑，有笔记本、台式机，装的window系统，花费了很多时间玩游戏。在工作之后，使用到mac电脑，感觉到一个完全不一样的世界。

mac电脑在我看来有众多优点。

  • 清爽，没有乱七八糟的垃圾软件、弹窗，不需要装杀毒软件，不用浪费精力时间在这些无谓的事情上，光这一点我认为就能值回价钱
  • 安静，正常情况下，除非运行比较消耗CPU的程序，mac基本不会有能听到的风扇声，对于噪音敏感的同学说一大福音，而我之前的神舟笔记本，那个风扇周期性的响声快把我烦死了。
  • 快捷键统一，在mac上的各个应用，常用快捷键都是一致的，比如关闭窗口、切换tab、打开配置等等。
  • 很多地方和linux类似，像做后台开发，比较熟悉linux系统命令，而在mac上，绝大部分命令都很类似，用起来很顺手
  • 不能玩游戏，很少有人用mac玩游戏，我认为这确实是一个很大的优点，如果能玩游戏，势必会更容易分心、被娱乐活动吸引

还有很多优点就不列举了，在我看来，mac上程序员等首选，建议大家早买早享受，真的。

mac电脑有多个系列，有macbook pro，macbook air, mac mini等，价格也有不同，一般建议买macbook pro，内存选择大一点，16G或更高。

macbook pro也就经常升级换代，一般来说买最新版本的就可以。

# iTerm

iTerm目前已经上mac默认的命令行程序了，不过默认的样式还是稍微朴素一点，一般我们都会安装上on-my-zsh让命令行看上去舒服一点

# on-my-zsh

解决githubusercontent.com访问比较慢的问题

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
1

# brew

brew上mac上安装软件的管理器，类似ubuntu上的apt、centos上的apm。通过brew安装软件，在大部分情况下都是比较方便的，一个brew install xxx命令即可完成，不需要关心安装步骤、依赖什么都。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
