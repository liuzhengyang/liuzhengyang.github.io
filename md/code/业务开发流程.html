# 开发流程

本文讲解一下典型的项目开发流程，对于大小项目都可以按照类似的步骤来执行。

作为业务开发，实现业务功能为用户带来价值，是实现自身价值的方式，所以做好功能开发是工作中最核心的要求。

# 需求分析

一般需求由产品经理提出，产品经理通过需求文档描述自己的需求，然后会拉会邀请研发（前端后端）、测试等同事参加需求评审。

在需求评审之前，建议提前看一下需求文档，提前对需求进行分析思考，思考合理性、实现成本，因为在会上可能进度比较快可能有问题不能及时 想到，将问题暴露的越早越好，避免开发过程中或之后才发现不合理。

分析需求，一看合理性，即这个需求是否合理，做完之后是否能达到希望的效果，实现成本如何，投入产出比多大。
