# InnoDB

# ACID

ACID 是数据库事务的四个关键属性，确保事务的可靠性和数据的一致性。

# Atomicity (原子性)

  • 定义：原子性指事务整体是一个不可分割的原子操作，事务中的操作要么全部完成，要么全部不执行。
  • 实现：
    • InnoDB 通过日志文件（redo log 和 undo log）来实现原子性。
    • 当事务提交时，InnoDB 会先将事务的操作记录到 redo log 中，确保数据的持久性。
    • 如果事务失败，InnoDB 会使用 undo log 来回滚事务，撤销已执行的操作。

# Consistency (一致性)

  • 定义：一致性保证事务的执行结果必须使数据库从一个一致状态转换到另一个一致状态，即事务执行前后，数据库的完整性约束不会被破坏。
  • 实现：
    • InnoDB 通过事务的隔离级别和约束检查来实现一致性。
    • 例如，如果一个系统要求用户的账户余额不能为负数，那么任何减少账户余额的操作都必须确保余额不低于零。

# Isolation (隔离性)

  • 定义：隔离性确保并发执行的多个事务之间互不影响。每个事务在执行过程中看起来就像是系统中唯一的活动。
  • 实现：
    • InnoDB 提供了不同的隔离级别，包括读未提交（Read Uncommitted）、读已提交（Read Committed）、可重复读（Repeatable Read）和序列化（Serializable）。
    • 隔离级别越高，并发度越低，但数据的一致性越好。
    • 例如，在可重复读级别下，事务可以多次读取同一数据，结果始终保持一致。

# Durability (持久性)

  • 定义：持久性指事务提交后，它对数据库的更改就是永久性的，不会因为系统故障而丢失。
  • 实现：
    • InnoDB 通过 redo log 来实现持久性。
    • 当事务提交时，InnoDB 会将事务的操作记录到 redo log 中，并确保这些记录被写入磁盘。
    • 即使系统发生故障，InnoDB 也可以通过 redo log 恢复未完成的事务，确保数据的完整性和一致性。

# 总结

通过上述四个属性，InnoDB 确保了事务的可靠性和数据的一致性。每个属性都有其特定的实现机制，共同协作以提供高性能和高可靠性的数据库服务。
