# Skywalking

# 介绍

skywalking是一款流行的开源(apache基金会）的apm（可观测性、监控系统），具备服务拓扑、metrics、tracing等观测功能，支持多种语音(java,go,js等)、支持丰富的框架(比如java中的常见中间件）。 skywalking生态活跃，开发者和使用者众多，还支持ebpf的持续性能收集、具备多种页面，是一个分布式tracing的好选择。

# 核心概念

  • Service. 服务，表示执行一定功能的一些资源的组合，比如登录服务、订单服务、评论服务、转码服务等。熟悉微服务的同学应该对服务概念比较了解。
  • Service Instance, 服务实例，一个服务中可能有多个实例（机器、容器等粒度）
  • Endpoint. 端点，比如接收http请求的uri path，像spring中的@RequestMapping
  • Process. 进程。在一些情况下，一个Service Instance不是一个进程，比如kubernetes 的pod可能包含多个container，container中可能包含多个进程

对于Java服务，skywalking提供了javaagent接入，用户只需要修改服务的启动参数，就可以自动集成skywalking，skywalking的 agent会通过字节码修改的方式修改要监控的框架代码，收集数据、发送到skywalking服务，用户通过skywalking的页面就能查看到服务的运行状态。

# 架构

  • Probe. 探针，比如Java中的javaagent，负责收集metrics、log、trace等数据并上报
  • Platform backend. Skywalking的服务端，负责接收Probe的数据、提供接口给UI查询数据。在skywalking还经常使用OAP(Observability Analysis Platform)表示backend。
  • Storage. 保存数据，skywalking支持丰富的数据源类型让用户自由选择，比如ElasticSearch, H2, MySQL, TiDB等等，还能够自行实现存储。
  • UI，页面。
