# 枚举的更新

# 新增枚举

通过前面的分析，新增枚举后，新的类编译后，javac会创建好对应的新字段，但是static代码块没有重新运行。

利用当前的class init plugin, class init plugin会创建一个$$ha$clinit静态方法。 $$ha$clinit方法会执行对原有类中不存在的static字段的赋值操作

# 修改枚举的非静态变量值

比如把 ITEM_1("I1") 改成 ITEM_1("I11")。 由于枚举字段对象是在static代码块中运行的，而static代码块没有重新运行。 所以需要重新执行全部的代码块。 重新执行静态代码块后，会重新创建每个枚举值的对象，所以如果外部有持有对象的引用缓存，则在使用==或equals判断时 会出现不符合预期的问题。
