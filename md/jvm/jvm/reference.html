# Reference

Reference: 在引用的对象被gc回收之后，gc会把对象放到ReferenceQueue中，程序能做相应的处理（从queue中poll，然后清理缓存等）。

另外通过WeakReference引用一些对象避免内存泄露也是Reference常见的使用方式。

Not Reference(Regular Class)

WeakReference: 弱引用，WeakReference引用不影响被引用的对象被gc回收，回收之后WeakReference.get会返回null SoftReference: 软引用，在需要回收时(发起concurrent marking时发现没有可用region也不能扩容heap），会通过LRU判断哪些对象需要回收。

All soft references to softly-reachable objects are guaranteed to have been cleared before the virtual machine throws an OutOfMemoryError. Otherwise no constraints are placed upon the time at which a soft reference will be cleared or the order in which a set of such references to different objects will be cleared. Virtual machine implementations are, however, encouraged to bias against clearing recently-created or recently-used soft references.

FinalReference：用来实现Finalizer功能，对象被回收后，如果对象实现了Object的finalize方法，则会执行该方法。此功能已经废弃不建议使用（可能出现性能问题、死锁等）。建议通过显示释放资源（比如AutoClosable）或ref.Cleaner、ref.PhantomReference等功能实现释放资源功能。 PhantomReference: 被回收后会被放到reference queue中，get方法永远返回null。 Cleaner: 被回收后直接调用clean方法。 Other: 其他继承Reference的类。

# Reference状态

  • Active: 会被GC特殊处理。此时referent != null，discovered = null or in GC discovered list.

  • Pending: pending-Reference 列表中的元素，等待被ReferenceHandler线程处理。referent = null; discovered = next element in pending-Reference list

  • Inactive: 不是Active和Pending的状态

  • Registered: 创建Reference对象时，和一个reference queue关联但是还没没有入队。此时queue=associated queue

  • Enqueued: 已经加到reference queue中，但是还没有出队。此时queue = ReferenceQueue.ENQUEUE; next = next entry in list or this to indicate end of list。

  • Dequeued: 已经从reference queue中出队。queue = ReferenceQueue.NULL; next = this

  • Unregistered: 创建Reference对象时没有和reference queue关联。queue = ReferenceQueue.NULL

GC只需要检查referent字段，discovered字段用来表示(非FinalReference类型)Reference对象需要特殊对待。 如果referent non-null并且不知道是否live，

# ReferenceHandler

ReferenceHandler是一个单独的线程，从ReferencePendingList中取出Reference，如果不是Cleaner，则加到ReferenceQueue中。 否则是Cleaner的话调用Cleaner的clean方法。 PendingList通过discovered字段当做next形成队列链表。 入队到ReferenceQueue中后，还会更新queue字段为ENQUEUED。(从ReferenceQueue中poll出队时，queue字段还会改为ReferenceQueue.NULL) ReferenceQueue通过Reference的next字段形成队列，出队后next指向自身。

class ReferenceHandler {
    public void run() {
        while (true) {
            processPendingReferences();
        }
    }

    private static void processPendingReferences() {
        // Only the singleton reference processing thread calls
        // waitForReferencePendingList() and getAndClearReferencePendingList().
        // These are separate operations to avoid a race with other threads
        // that are calling waitForReferenceProcessing().
        waitForReferencePendingList();
        Reference<?> pendingList;
        synchronized (processPendingLock) {
            pendingList = getAndClearReferencePendingList();
            processPendingActive = true;
        }
        while (pendingList != null) {
            Reference<?> ref = pendingList;
            pendingList = ref.discovered;
            ref.discovered = null;

            if (ref instanceof Cleaner) {
                ((Cleaner)ref).clean();
                // Notify any waiters that progress has been made.
                // This improves latency for nio.Bits waiters, which
                // are the only important ones.
                synchronized (processPendingLock) {
                    processPendingLock.notifyAll();
                }
            } else {
                ref.enqueueFromPending();
            }
        }
        // Notify any waiters of completion of current round.
        synchronized (processPendingLock) {
            processPendingActive = false;
            processPendingLock.notifyAll();
        }
    }
}

class ReferenceQueue {
    boolean enqueue(Reference<? extends T> r) { /* Called only by Reference class */
        synchronized (lock) {
            // Check that since getting the lock this reference hasn't already been
            // enqueued (and even then removed)
            ReferenceQueue<?> queue = r.queue;
            if ((queue == NULL) || (queue == ENQUEUED)) {
                return false;
            }
            assert queue == this;
            // Self-loop end, so if a FinalReference it remains inactive.
            r.next = (head == null) ? r : head;
            head = r;
            queueLength++;
            // Update r.queue *after* adding to list, to avoid race
            // with concurrent enqueued checks and fast-path poll().
            // Volatiles ensure ordering.
            r.queue = ENQUEUED;
            if (r instanceof FinalReference) {
                VM.addFinalRefCount(1);
            }
            lock.notifyAll();
            return true;
        }
    }
}

# PhantomReference和Cleaner

PhantomReference的get方法固定返回null。 通过Cleaner类，可以实现对象被回收时执行回调方法。

public class Cleaner
    extends PhantomReference<Object> {

    // Dummy reference queue, needed because the PhantomReference constructor
    // insists that we pass a queue.  Nothing will ever be placed on this queue
    // since the reference handler invokes cleaners explicitly.
    //
    private static final ReferenceQueue<Object> dummyQueue = new ReferenceQueue<>();

    // Doubly-linked list of live cleaners, which prevents the cleaners
    // themselves from being GC'd before their referents
    //
    private static Cleaner first = null;

    private Cleaner
            next = null,
            prev = null;

    private static synchronized Cleaner add(Cleaner cl) {
        if (first != null) {
            cl.next = first;
            first.prev = cl;
        }
        first = cl;
        return cl;
    }

    private static synchronized boolean remove(Cleaner cl) {

        // If already removed, do nothing
        if (cl.next == cl)
            return false;

        // Update list
        if (first == cl) {
            if (cl.next != null)
                first = cl.next;
            else
                first = cl.prev;
        }
        if (cl.next != null)
            cl.next.prev = cl.prev;
        if (cl.prev != null)
            cl.prev.next = cl.next;

        // Indicate removal by pointing the cleaner to itself
        cl.next = cl;
        cl.prev = cl;
        return true;

    }

    private final Runnable thunk;

    private Cleaner(Object referent, Runnable thunk) {
        super(referent, dummyQueue);
        this.thunk = thunk;
    }

    /**
     * Creates a new cleaner.
     *
     * @param  ob the referent object to be cleaned
     * @param  thunk
     *         The cleanup code to be run when the cleaner is invoked.  The
     *         cleanup code is run directly from the reference-handler thread,
     *         so it should be as simple and straightforward as possible.
     *
     * @return The new cleaner
     */
    public static Cleaner create(Object ob, Runnable thunk) {
        if (thunk == null)
            return null;
        return add(new Cleaner(ob, thunk));
    }

    /**
     * Runs this cleaner, if it has not been run before.
     */
    @SuppressWarnings("removal")
    public void clean() {
        if (!remove(this))
            return;
        try {
            thunk.run();
        } catch (final Throwable x) {
            AccessController.doPrivileged(new PrivilegedAction<>() {
                public Void run() {
                    if (System.err != null)
                        new Error("Cleaner terminated abnormally", x)
                                .printStackTrace();
                    System.exit(1);
                    return null;
                }
            });
        }
    }
}

# SoftReference

SoftReference的清理策略，是根据当前内存的剩余情况来判断的，如果剩余内存比较多，则会留下更多的SoftReference的referent引用， 否则更少，在FullGC时会全部清理。 不是FullGC时，会通过上次gc剩余的内存空间(MB单位）乘以SoftRefLRUPolicyMSPerMB配置参数（默认1000ms）作为SoftReference引用的 对象是否被回收的判断条件，如果SoftReference最近使用时间(get时会更新)距离当前时间(gc维护的clock时间)大于这个阈值，则被清理，否则不被清理。

静态的clock值会在gc时由jvm更新，来提高性能。

public class SoftReference<T> extends Reference<T> {

    /**
     * Timestamp clock, updated by the garbage collector
     */
    private static long clock;

    /**
     * Timestamp updated by each invocation of the get method.  The VM may use
     * this field when selecting soft references to be cleared, but it is not
     * required to do so.
     */
    private long timestamp;

    /**
     * Creates a new soft reference that refers to the given object.  The new
     * reference is not registered with any queue.
     *
     * @param referent object the new soft reference will refer to
     */
    public SoftReference(T referent) {
        super(referent);
        this.timestamp = clock;
    }

    /**
     * Creates a new soft reference that refers to the given object and is
     * registered with the given queue.
     *
     * @param referent object the new soft reference will refer to
     * @param q the queue with which the reference is to be registered,
     *          or {@code null} if registration is not required
     *
     */
    public SoftReference(T referent, ReferenceQueue<? super T> q) {
        super(referent, q);
        this.timestamp = clock;
    }

    /**
     * Returns this reference object's referent.  If this reference object has
     * been cleared, either by the program or by the garbage collector, then
     * this method returns {@code null}.
     *
     * @return   The object to which this reference refers, or
     *           {@code null} if this reference object has been cleared
     */
    public T get() {
        T o = super.get();
        if (o != null && this.timestamp != clock)
            this.timestamp = clock;
        return o;
    }

}
