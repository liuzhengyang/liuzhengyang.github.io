# WispEventPump

EventPump名称叫事件抽水泵，用来观察有哪些事件发生。 默认每8个worker创建一个EventPump io有读写时间时唤醒wisp carrier thread执行对应的wisp task（协程）

在协程阻塞等待io事件时，会通过WispEventPump.Poll.INSTANCE.registerEvent来注册事件，当fd事件发生时，WispEventPump会通过processEvent来 找到监听该事件的WispTask，调用WispTask.jdkUnpark唤醒。

# JDK java代码的修改

线程阻塞调用，比如Lock，会调用AQS的tryAcquire，如果没有获取成功需要等待，会调用LockSupport.park，LockSupport又调用Unsafe.park。 wisp修改了Unsafe类。同理线程恢复的unpark也是做了类似拦截。

class Unsafe {
    public void park(boolean isAbsolute, long time) {
        if (WispEngine.transparentWispSwitch()) {
            if (time <= 0) { // non-timeouted park
                SharedSecrets.getWispEngineAccess().park(0);
            }
            long timeout = !isAbsolute ? time:
                    java.util.concurrent.TimeUnit.MILLISECONDS.toNanos(time - System.currentTimeMillis());
            if (timeout > 0) {
                SharedSecrets.getWispEngineAccess().park(timeout);
            }
            return;
        }
        park0(isAbsolute, time);
    }
    public void unpark(Object thread) {
        if (WispEngine.transparentWispSwitch()) {
            if (thread instanceof Thread) {
                SharedSecrets.getWispEngineAccess().unpark(
                        SharedSecrets.getJavaLangAccess().getWispTask((Thread) thread));
            }
            return;
        }
        unpark0(thread);
    }
}
