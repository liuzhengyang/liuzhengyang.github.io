# EPoll

# EPoll介绍

EPoll是linux提供的网络事件通知框架。 在网络交互中，代码关心的是，什么时候有数据可读（对方发送了请求或对方请求返回数据返回了）， 什么时候可以写数据，有哪些客户端建连。

每个连接在linux中通过fd(file descriptor)表示，我们需要监听各个fd的可读可写事件。 在早期linux提供的是poll的机制，缺点是随着要监听的fd数量变多，poll的遍历开销越来越大。 epoll是全新的扩展性更好的实现，不会因为fd增多而性能下降。

epoll API的核心概念是epoll实例。通过API epoll_create可以创建epoll实例，epoll实例表示一个内核数据结构， 在用户态视角，可以把它想象成两个list的容器。

  • interest list : 包含的是对哪些fd(连接）的那些事件（可读、可写、可accept等）感兴趣
  • ready list: 哪些fd的IO事件准备好了。ready list是interest list的子集。ready list是内核根据IO活动变化动态维护的。

应用层可以通过如下的系统调用来使用epoll

  • epoll_create : 创建一个新的epoll实例，返回指向这个实例的fd
  • epoll_ctl : 通过epoll_ctl能够添加、修改、删除感兴趣的fd事件
  • epoll_wait : 等待I/O事件，没有I/O事件时会阻塞调用线程，直到有I/O事件或超过传入的超时时间

edge-triggered 和 level-triggered, 默认是level-triggered模式，edge-triggered模式下，如果出现可读事件， epoll_wait会返回，但是后面再收到的数据调用epoll_wait不能获取到可读事件，level-triggered模式没有这个问题。
