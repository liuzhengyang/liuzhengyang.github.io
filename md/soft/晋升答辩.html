# 像准备比赛一样准备晋升答辩

同学你好，我是bytejava刘老师，今天分享一些关于晋升答辩的心得经验，包含了针对晋升答辩的分析、答辩如何准备、ppt注意事项等，希望能给有需要的同学一些帮助。

我在多家大型互联网公司参加过多次晋升，次数和成功率应该都是比较高的，在我看来，晋升答辩是一项比赛，就像编程比赛、跑步比赛、赛车比赛、游戏比赛一样， 准备比赛要讲究知识技巧（按照互联网的黑话说是「打法」「方法论」），这样经过科学的努力准备、训练之后，相信你也能比别人具备更大优势取得更好的成绩。

# 为什么会有职级晋升

职级晋升是大部分互联网公司都会有的流程，一般会每年进行。

不同的工作方向比如开发、产品、测试、运营都有对应的晋升，本篇文章主要针对技术（服务端），但是其中的思想我想是通用的。

有职级晋升的前提是有职级体系。很多公司都会自创一套体系，比如KX, PX, TX, DX， 后面的X是数字表示的级别高低，用来对应工程师的级别，比如初级工程师、高级工程师、技术专家、高级技术专家等等。

职级体系的目的有很多，核心的目的是方便进行员工管理。

  1. 招聘面试时，面试官给候选人评估职级，然后HR按照对应的职级评估薪资。
  2. 员工工作时，会按照晋升的职级标准进行努力，激励员工做出更多贡献，从而可以引导员工向企业需要的方向发展。
  3. 职位（比如XX总监、XX负责人）和职级一般有范围约束关系，职位越高需要的职级也越高

对于员工而言，晋升意味着薪资待遇提升和能力认可。 同时职级是身份的象征，级别越高，表示专业能力、综合能力越强，是个人的身份标签，比如面试时了解对方是XXX腾讯T几、阿里P几或者说职位比如负责人、总监等，就能让别人对他的能力水平有一个初步地了解。

好了，总结来说，职级晋升能让员工为了升职加薪工作更加努力，为公司创造更大价值。

# 如何准备职级晋升

晋升一般会采取答辩的形式，分为答辩人和评委，答辩人一般通过PPT的形式向评委展示自己的工作能力，展示完成后，评委询问问题，然后评委 对答辩人进行评价打分，最终打分结果也决定了晋升能否通过。

晋升通过的逻辑，晋升通过，表示评委认可你的能力达到了「目标职级的要求」并且（如果有通过率、通过人数要求）在答辩人当前位于前列。

这里目标职级的要求非常重要，每个岗位方向都会有各自的能力要求，一般公司也会给出能力说明，所以在答辩时要注重展现自己在这些方面的能力，就像写作文要命中主题一样。

# 晋升答辩的准备

晋升答辩一般越早准备越好，提前了解目标职级的能力要求，然后在工作中，主动去锻炼自己对应的能力。 比如目标职级要求有独立负责一个项目的能力，则自己要争取能够独立负责一个项目。

另外要主动多做一些有挑战性的项目、任务，这些有挑战性的工作是未来晋升答辩时的重要能力证明。

有的同学可能会说，我当前的工作都很简单，没有什么高并发、复杂度，我该怎么办呢？

这样的情况有几种方法。

  • 主动找活，在比较好地完成已有工作的情况下，主动向领导申请承担更大的项目
  • 深挖，针对项目中出现的问题，深入挖掘问题出现原因、解决方案等，比如你遇到一个异常，不断深入研究，发现是开源框架的实现问题，你修复并合并到了对应的开源项目中，这就是一个比较好的例子。
  • 注重业务上的思考，思考系统如何设计，具备扩展性，更好更快地支持产品的需求

好，现在到了公司答辩的时候，一般会提前一个月通知大家，这时候如果你确定要参加答辩，就要开始正式准备了。 准备答辩尽量放在工作之余比如周末或下班后、中午休息时，避免占用太多工作时间。

# PPT内容结构

有的公司会给出答辩PPT模板，可以进行参考。

PPT的整体要求结构感，参考《金字塔原理》，内容要讲究有层次结构、以上统下、各个部分有逻辑关系。

答辩PPT的整体目标就是证明你是优秀的员工、具备目标职级的能力、具备成长潜力。 然后在部分上，分为几个部分。这里给出其中一个结构实例。 分为四部分

  • 第一部分个人介绍，概述性的介绍自己过去的成长经历，包括负责的项目、角色，取得的成绩结果等。
  • 第二部分找几个重点的项目介绍。重点项目中也有子结构，比如项目的背景、难点挑战、你在其中的角色职责、做了哪些努力如何实现的，最终取得了什么样的效果，你做的和行业上其他公司做的有哪些优势，做完了之后你总结了一套方法论，能够把经验复制到其他的项目中。
  • 第三部分是个人的其他素质展示，比如有带人的经验总结、过去的技术分享等影响力等
  • 最后一部分，未来规划，未来规划向评委展示自己的规划能力，并且能够让评委看到你未来能给公司带来的价值。

搭建好结构后就是准备内容了，将自己过去的项目、工作，找出符合职级要求的材料，准备到一个文档中。然后不断抽象，写PPT。

# 晋升答辩和年终总结的区别

很多同学会把自己做的大大小小的项目，不管重不重要都罗列到ppt中。 罗列项目是工作汇报的重点，所以一定要把晋升和工作汇报区分开。

年终总结是对过去一年项目的回顾，重点是项目的完成情况的总结，完成的比较好比如完成已有计划并且超出预期，则会在年终考核时具备优势。

晋升答辩更加注重展示个人而不是事情，答辩时展示项目的目的是证明人具备了哪些能力、思考、潜力。

潜力这一点，其实非常重要，根据心理学原理，人们更愿意招聘提拔更有潜力的员工，你如果能通过答辩内容，展示出你强大的学习进步能力，展示出你未来更给公司带来的无限可能， 相信评委更愿意投你一票。

# PPT的图表、格式

写好PPT确实非常重要，因为即使相同的内容，PPT做的好和做的不好有天壤之别。 大家可能会说我不会写好的ppt，确实大部分人都没有训练过ppt，刚开始写出来的ppt都会很简陋。 不过这也就有了优势，如果你稍微比别人努力一点，把ppt做好一点，就能给评委留下更好的印象，就像 高考阅卷老师看到好的字体一样。

关于ppt，建议平时有时间的时候多注意观察练习，比如找一些ppt的模板网站学习、找一些大会分享的ppt材料学习， 找出他们做的好的ppt结构、图标，进行模仿练习，如果你更狠一点，可以学习专门的ppt课程，掌握ppt能力后再回到 互联网公司答辩绝对是降维打击，有了这项技能，未来的工作汇报也是一大优势。

PPT非常重要的一点是要少字、多图，不要大段大段编写文字。

因为晋升答辩，是看个人的业绩、能力是否匹配目标职级的要求。 所以呢，晋升的内容要注重以展现个人的能力为目标，每个内容都要有目的，不是周报、年终总结，不要记流水账。

比如写一个项目，要写上问题背景，你针对这个问题的分析过程，挑战难点，解决的过程，做完的效果，最好也能写上和行业上其他公司的做的对比优势。 然后呢，最好还要有一个思考总结，从中总结出一些方法论（方法论就是做事的通用做法，即你掌握了一套做事的方法，类似的事情也能通过相应的方法解决）

所以，重要的还是要在晋升前的一两年，做一些能够拿得出手的项目，会更好通过，平时有好的项目机会也要积极争取。

# 演练

演练的目的是提前发现问题、控制时间、充分准备克服紧张情绪。

内容准备好之后，很重要的一个环节就是演练了，演练非常重要，一定要重视。 很多人会说老师，答辩的时候我好紧张。可以说，紧张就是因为准备不足，准备不足所以不自信，不自信又导致紧张， 紧张忘词又更紧张恶性循环。所以一定要频繁充分的演练练习，你把内容牢牢的掌握之后，会更加自信、讲起来的感觉也完全不一样。

另外自己演练讲出来的时候，会发现很多自己写的文字读起来不通顺，所以内容还需要不断改进。

时间控制也是非常重要的一方面，因为答辩一般都有时间控制，建议时间控制在比要求的时间稍微短一点。

最后非常建议，进行模拟答辩，请组里、部门的资深人员帮你模拟答辩，他们有更丰富的经验、并且站在评委的角度也能 发现你自己意识不到的很多问题，这一点非常重要。

# 关于答辩紧张问题

很多人在当众讲话时会紧张，在关键场合尤其像晋升这种涉及到自己的薪资待遇的场合更会紧张，紧张的影响是 会影响自己的发挥，并且太紧张时展现出来的答辩效果也不好，给评委留下不自信、不好的印象。

这里我有一些心得和大家分享。

  • 看淡点，晋升成功，也只是加一点工资，而这点工资，即使这次没加上，后面的涨薪也会加上，或者跳槽涨得比这个多多了。
  • 把思考的时间拉长一点，拉长到10年、20年甚至整个职业生涯，一次晋升的结果，没有那么重要。再过一百年，大家还都不存在了呢，不要太在意。
  • 多练习，其实紧张的很大原因是自己准备不足，准备不足就会，如果你准备的非常充分，每天演练几遍，非常熟练后，答辩演讲时说的话也会很自然自信。
  • 不要怕评委，评委也都是普通人，虽然比你职级高，经验丰富，不过你们处于不同的成长阶段，没有必要妄自菲薄，自信一点。
  • 把晋升当做一次成长的机会，平时工作一直在干活，晋升算是一次很好地总结梳理的机会，从晋升的标准要求，也能发现自己的不足、未来要注重培养的方向。
  • 准备时，是一定要多大声演练的，不要心里准备，文字有的写的也不通顺，因为很多话说出来才会发现之前的文字不通顺。

总结就是，频繁练习、熟能生巧、生死看淡。

# 关于答辩评委问题

个人陈述完了之后呢，评委就会问你问题了，通过问题，来更加详细的评估你的能力。

对于问答环节，我认为最重要的一点是要确认好评委的问题，避免出现答非所问的情况，并且回答要直面问题，不要回答半天，回答不在重点。 最好要提前自己思考有哪些问题评委可能会问到，或者请更资深的员工帮自己看。 如果遇到不会的问题或做的不好的地方，要如实积极的回答，比如评委发现你有一个地方设计的、做的不好，尽量大方承认并且表示未来会优化改进，我认为还是尽量不要撒谎， 一方面评委会看出来，一方面不要为了一点利益损失个人品格。

# 晋升结束之后

晋升如果成功，说明职级得到提升薪资待遇一般也有有所提升，当然是好事，不过要注意职别升高之后，对于你的要求也变高了，要开始瞄准下一个级别的要求去工作了，自己的压力和责任也会变大。 如果晋升没成功，不要太伤心，吸收一下反馈和教训，可以继续努力，下次再来，或者努力跳槽，跳槽时你就可以获得更高的级别和涨幅。

晋升就像排名比赛，有人靠前成功有人靠后失败。 没通过自然心里是不好受的，所以如果要参考，一定要多提前努力做好准备，那假如还是没通过（概率还挺高）， 有的人适合越挫越勇，杀不死我的会让我更强大，从答辩结果继续努力或者跳槽到更厉害的公司的更高级别，从前的我你爱理不理，现在的我你高攀不起。 有的人佛系一点的，需要调节下情绪，你要想，你还可以通过明年的加薪来涨工资呢，结果不也差不多吗，别人就是提前加了一点工资而已，互联网能不能干到35岁还不一定呢，人又能活几年呢， 或许能提醒多发展一些其他挣钱技能，在其他人一辈子打工的时候，你的未来有更大的潜力。

# 晋升答辩与面试的区别

大家不要抱怨说怎么新招进来的人这么水级别还这么高，因为通过招聘进到公司的员工是能拿到更多的溢价的，综合来看晋升的难度要比面试要高，我这里给出一些个人的分析。

  1. 面试时，面试官作为业务方，是更加期望能够招到人的，并且招到更多人能够让面试官管理更多员工更有利于他管理权利的增加管理职级增加，并且在企业扩张时，招人的成本也不会对业务负责人有太多影响
  2. 面试时，候选人可以拿到其他offer作为竞争条件，如果企业给不到有竞争力的薪资，候选人是不会来的，所以候选人一般会选择最有竞争力的offer，这样就会形成，面试进来的员工，高级别看上去很好拿到（相对内部晋升）
  3. 内部晋升时，你晋升通过，对于评委是没有太多好处的，而且很多评委都是其他部门的，出现跨部门评审，或多或少都会打压其他部门的同事。
  4. 内部晋升即使不通过，很多员工也没有任何办法，不像面试一样你不给我对应职级我就不来了，不是所有员工都会一怒换工作，所以这里也有一个换工作带来的成本（准备成本、更换环境相对熟悉环境的压力）会对晋升人员产生影响，公司就有更多议价空间。

# 总结

总结呢，晋升答辩确实是一个比较关键的事情，但是也不要过度重视，放平心态，多练习，把它当成一次总结成长的机会。 如果结果很好，继续努力；如果结果不好，看淡一点，说不定有更好的机会呢。
