# The Manager's Path 管理者之路

# 前言

在互联网公司中，一般按照树状结构进行部门组织划分，树的叶子节点上最基层 的普通员工，其他的非叶子节点则承担管理职责，负责管理下面的员工、组织。

我待过两家比较大的互联网公司，目前工作8年左右，期间更换过10多个管理者， 中间也虚线带过一段时间其他人。 所以我有比较丰富的被管理的经历，每个管理者都有自己的特点、管理风格，我也对行业的现状比较感叹，确实很少能有优秀或者说合格的管理者。 作者也提到很多人可能在整个职业生涯过程中，一个好的管理者也遇不到。 好的管理者为团队成员的成长着想、带领团队完成任务、成就他人、团队环境气氛好，不好的管理者有的放任不管、有的人品性格有缺陷打压别人以个人利益为中心。 同时回首我带人的经历，很多地方也做的不好。 我们还是要积极面对这个环境，这本书对于个人的意义在于，识别什么是好的管理者什么是不好的管理者，我们无力改变环境改变别人，那就在有选择能力时跟随对的人， 同时本书也指明了管理者的成长路径，清晰地指出该做什么该怎么做，有了这些知识准备，相信在有承担管理职责的机会来临时，我们能够更好地把握住机会。

所以希望自己能够掌握本书内容，在未来有机会时，更好地通过管理帮助别人帮助团队，同时也成就自己。

# One-on-one meetings

One-on-one meetings是指定期的一对一会议。然而很多管理者忽视这些会议，或者让他们感觉是在浪费你的时间。

(ps: one-on-one目前看上非常重要的管理方法，不过互联网环境下能够定期和团队成员进行一对一会议的管理者可能很少，就我个人而言，从业8年10多个领导，还没有进行任何一次一对一会议， 观察来看，进行过一对一的团队也是很少的)

one-on-one有两个目的，首选，一对一会议创建你和管理者之间的人类之间的连接，能让彼此更加了解，了解除了 工作之外的比如生活的事情，让我们能意识到大家都是活生生的人，而不是没有情感的工作机器。强大的团队的基石上人与人之间的连接， 人和人的连接带来信任，信任上非常重要的事情。 第二个目的是，给你一个固定的可以和管理者讨论需要私下谈论的内容的机会。

尽量不好把这个会议做成工作汇报，工作汇报通过其他形式，比如日会周会进行，one-on-one更加进行工作内容之外的讨论， 比如职业规划、人生计划、生活困扰、兴趣爱好等都是比较好的话题。作为被管理者，我们要提前准备好时间、准备好像讨论的内容。

# 反馈和工作指导

第二个我们期望从管理者获得的是反馈，不仅仅是绩效review。因为每个人都不可避免的会做错一些事情，如果 你的管理者合格的话，他就应该及时让你知道你哪里做的不好。收到这样的反馈难免会让人感到不舒服，尤其是 刚刚加入职场的新人，因为之前只有父母会给自己行为上的反馈。 但是我们都是需要这样的反馈的，就像在海上航行能及时知道有没有走偏。不好的管理者会不给你反馈或只有在绩效review的时候 才告诉你哪里不好，越早知道自己的不好的习惯，就越容易改正。 反馈不仅仅指错误纠正，鼓励也是非常重要的，好的管理者能够注意到你做的好的地方并鼓励你保持。

理想情况下，管理者应该公开表扬、私下批评。

还有其他的可以从管理者得到的反馈，比如你在准备ppt，可以让管理者帮忙review给出建议，因为他们有更多经验和不同的视角。

同时，向别人寻求建议，也是一种表达尊重的很好的方式。人们都喜欢感觉自己对别人有用有帮助， 管理者对这种讨好，也是不会拒绝的。

如果公司有职级，要和你的管理者询问如果你想晋升该关注提升哪些方面的能力。 如果你和同事或其他团队合作遇到困难，你的管理者应该帮助你寻找解决方案，必要时能出面帮你协调解决。

好的管理者能够给你合适的项目帮助你成长进步，好的管理者能够帮助你理解你在做的工作的价值，给你展示你在 团队的目标的角色，让你每天的工作有目标感，当你明白你做的工作对于公司的整体成功有贡献时，你会更加自豪。

# 培训、职业成长

在公司这样的层级体系下，你的管理者上有责任帮助你提升的，通过帮助你找到培训或其他成长的资源。 比如可能是告诉你需要的书籍，指出需要学习的领域等。 如果有晋升体系，管理者应该辅导你晋升。
